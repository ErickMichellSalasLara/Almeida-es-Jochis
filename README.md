# Almeida-es-Jochis

 - Ocupamos un sistema de Puntuacion por cada ronda que se gane n el juego
 - Un menu donde se pueda dar a "Jugar"
 - Los "pokemones" los cuales se haran e una herencia
 - Encapsulamiento de los valores en cada funcion
 - Sistema de combates por turnos
 - Selección de ataques de los pokemones (Cada uno con 4 ataques)
 - Sistema de conters entre tipos pokemon ( Agua >>> Fuego ) pero con tipos personalizados de la uni
 - Un maximo de 10 pokemones y 5 entrenadores estaria bien
 - Sistema de game over
 - Seleccion individual o random de pokemones
 - Que el juego pida el nombre de la persona que jugara para asi poder referirse a el/ella
 - Sonidos (En general)
